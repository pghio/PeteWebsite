# Security policy

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability or exposed secret. Email `pmghiorse@gmail.com` with the affected URL or file, a concise reproduction, and the potential impact.

This is a static personal site. It does not accept passwords, payment data, or visitor-provided profile information. Newsletter subscriptions are handled by Substack. Aggregate analytics are consent-gated, and person-associated outreach links must never place a name or email address in analytics or a public URL.

## Supported version

Only the production version at `peterghiorse.com` and the current default branch receive fixes.
